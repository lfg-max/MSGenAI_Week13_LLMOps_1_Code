📖 What This App Does (In Simple Terms)

Imagine you have a massive spreadsheet with hundreds of thousands of rows of supply chain data - orders, shipments, sales, profits, and more. Normally, you'd need to know complex data analysis tools like Excel, SQL, or Python to get answers from this data.

This application solves that problem by letting you ask questions in plain English, just like you'd ask a human colleague. You can type things like:

  • "What is the total sales?"
  • "Which shipping mode has the highest profit?"
  • "Show me a chart of sales over time"

Here's how it works behind the scenes:

  • AI Understanding: The app uses AI (GPT) to understand your question in plain English
  • Smart Code Generation: It automatically writes the necessary code (Python or SQL) to analyze your data
  • Safety Check: Another AI reviews the generated code to make sure it's safe and correct before running it
  • Execution: The approved code runs on your data and produces results
  • Context Awareness: The app also retrieves relevant business knowledge (like what "late delivery risk" means) to give you better, more informed answers
  • Natural Response: Finally, it explains the results back to you in plain English, and can even create charts and graphs

Think of it as having a super-smart data analyst who knows your supply chain business inside and out, can instantly write complex analysis code, and explains everything in simple language - all through a chat interface.

Data Files Used Throughout Execution

  • 95MB supply chain transaction dataset

📋 Complete File Call Summary (Per Query)

  • app.py - User input → Result display
  • config.py - Configuration loading (once at startup)
  • chain.py - Main orchestrator (all pipeline steps)
  • ingestion.py - Data loading (once at first query)
  • vector_store.py - Vector retrieval (every query)
  • sql_tools.py - SQL execution (if SQL mode)
  • intent_classification.prompty - Intent classification (every query)
  • code_generation.prompty OR sql_generation.prompty - Code generation (every query)
  • code_evaluation.prompty - LLM judge (every query)
  • response_synthesis.prompty - Response synthesis (every query)
  • DataCoSupplyChainDataset.csv - Data source (loaded once)
  • domain_knowledge.txt - RAG context (indexed once, retrieved every query)
